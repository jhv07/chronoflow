CHRONOFLOW - FULL-STACK TIMELINE APPLICATION
RUN INSTRUCTIONS
================================================================================

PREREQUISITES:
--------------
1. Python 3.8 or higher installed
2. MongoDB installed and running (local instance or MongoDB Atlas)
3. pip package manager
4. A modern web browser (Chrome, Firefox, Edge, Safari)

STEP 1: INSTALL MONGODB
-----------------------
Option A - Local MongoDB:
    - Download and install MongoDB Community Server from:
      https://www.mongodb.com/try/download/community
    - Start MongoDB service:
      Windows: MongoDB should start automatically as a service
      Mac/Linux: Run: sudo systemctl start mongod (or mongod command)

Option B - MongoDB Atlas (Cloud):
    - Sign up at https://www.mongodb.com/cloud/atlas
    - Create a free cluster
    - Get your connection string
    - Update MONGO_URI in config.py

STEP 2: SETUP PYTHON ENVIRONMENT
--------------------------------
1. Open terminal/command prompt in the project directory:
   cd C:\Users\Gnana\Desktop\final-chronoflow

2. Create a virtual environment (recommended):
   python -m venv venv

3. Activate virtual environment:
   Windows: venv\Scripts\activate
   Mac/Linux: source venv/bin/activate

4. Install required packages:
   pip install -r requirements.txt

STEP 3: CONFIGURE MONGODB CONNECTION
-------------------------------------
1. Open config.py
2. Update MONGO_URI if using MongoDB Atlas:
   MONGO_URI = 'mongodb+srv://username:password@cluster.mongodb.net/chronoflow'
   
   OR for local MongoDB:
   MONGO_URI = 'mongodb://localhost:27017/chronoflow'

3. (Optional) Update JWT_SECRET_KEY for production:
   JWT_SECRET_KEY = 'your-strong-secret-key-here'

STEP 4: CREATE REQUIRED DIRECTORIES
------------------------------------
The application will automatically create:
- static/uploads/ (for event photos)

If needed, create manually:
   Windows: mkdir static\uploads
   Mac/Linux: mkdir -p static/uploads

STEP 5: START THE APPLICATION
------------------------------
1. Make sure MongoDB is running (check with MongoDB Compass or mongosh)

2. Start the Flask server:
   python app.py

TROUBLESHOOTING
---------------
Problem: "MongoDB connection failed"
Solution:
  - Ensure MongoDB is running
  - Check MONGO_URI in config.py
  - For MongoDB Atlas: Check network access and credentials

Problem: "Port 5000 already in use"
Solution:
  - Change PORT in config.py to another port (e.g., 5001)
  - Or stop the process using port 5000

Problem: "Module not found"
Solution:
  - Ensure virtual environment is activated
  - Run: pip install -r requirements.txt

Problem: "Sound notifications not working"
Solution:
  - Allow browser notification permissions
  - Check browser settings for sound/audio permissions
  - Service Worker should be registered automatically

Problem: "Events not saving"
Solution:
  - Check MongoDB connection
  - Verify you're logged in (check for JWT token in localStorage)
  - Check browser console for errors

DATABASE COLLECTIONS
--------------------
The application creates these collections in MongoDB:
- users: User accounts (username, email, password hash)
- events: Timeline events (date, time, title, description, category, etc.)
- user_settings: User preferences (theme, background, notifications)

VIEWING DATA IN MONGODB COMPASS
--------------------------------
1. Open MongoDB Compass
2. Connect to: mongodb://localhost:27017
3. Select database: chronoflow
4. Browse collections:
   - users: See all registered users
   - events: See all events with full details including triggered status
   - user_settings: See user preferences
